# 13_DEVELOPMENT_PLAN：开发计划与分工

## 1. 开发阶段

### 第一阶段：基础工程搭建

目标：项目可启动。

任务：

- 初始化 Spring Boot 后端；
- 初始化 Next.js 前端；
- 配置 MySQL、Redis；
- 配置 Flyway；
- 创建基础表；
- 实现统一响应和异常处理；
- 实现登录注册。

### 第二阶段：航班查询与后台管理

目标：航班数据可维护、可查询。

任务：

- 航司管理；
- 机场管理；
- 航班管理；
- 航班查询接口；
- 航班筛选页；
- 航班详情页。

### 第三阶段：订票、选座和订单

目标：用户可以完成订票。

任务：

- 座位生成；
- 座位图接口；
- 乘机人管理；
- 创建订单；
- 锁定座位；
- 模拟支付；
- 我的订单页面。

### 第四阶段：退票和候补

目标：完成推荐完整版本的业务闭环。

任务：

- 退票规则；
- 退票释放座位；
- 候补订单；
- 退票后候补触发；
- 候补支付与过期释放。

### 第五阶段：AI 智能购票助手

目标：完成推荐完整版本的智能推荐能力。

任务：

- AI 聊天页面；
- 规则解析；
- 缺失字段追问；
- 查询真实航班；
- 推荐卡片；
- 跳转链接。

### 第六阶段：测试、部署、文档和展示

目标：可交付。

任务：

- 接口测试；
- 并发订票测试；
- 修复 Bug；
- 完成部署；
- 录制系统视频；
- 制作 PPT；
- 完成实训报告。

## 2. 三人分工建议

### 成员 A：后端核心业务

负责：

- 用户认证；
- 航班查询；
- 座位库存；
- 订单；
- 退改签；
- 候补；
- MyBatis SQL。

### 成员 B：前端用户端

负责：

- 首页；
- 航班查询页；
- 航班详情页；
- 订票选座页；
- 我的订单页；
- AI 助手页面。

### 成员 C：后台、数据库、测试和文档

负责：

- 管理后台；
- Flyway 脚本；
- 演示数据；
- 部署；
- 测试；
- PPT 和报告整理。

## 3. 最小可交付版本 MVP

必须完成：

- 登录注册；
- 航班查询；
- 航班详情；
- 选座；
- 创建订单；
- 模拟支付；
- 我的订单；
- 管理员航班管理；
- Flyway 初始化数据库。

MVP 不要求退票、候补、AI 和改签，但必须把订单、座位和余票的一致性做好，否则后续扩展会返工。

## 4. 推荐完整版本

在 MVP 基础上增加：

- 高级筛选；
- 退票；
- 候补；
- AI 智能购票助手；
- 数据统计；
- Docker Compose 部署。

## 5. 加分版本

继续增加：

- 改签；
- LLM 意图解析；
- JMeter 并发测试报告；
- 服务器部署；
- 更完整的数据报表。
