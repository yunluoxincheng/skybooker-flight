# 已知缺陷清单

当前清单来自用户交付前测试准备阶段反馈。2026-07-07 已完成 KI-001 至 KI-006 的浏览器复现抽样，并已为确认缺陷创建 GitHub Issue。未覆盖到的同类页面仍需后续扩展验证，不得在未复测前标记为已修复或通过。

## 严重级别

| 级别 | 标准 |
|---|---|
| S0 阻塞 | 系统无法启动、主流程完全不可用、无法登录、无法下单、严重数据损坏 |
| S1 严重 | 核心功能错误、安全越权、支付/订单状态错误、用户数据泄露 |
| S2 一般 | 非核心功能异常、部分筛选错误、提示不准确、普通页面交互问题 |
| S3 轻微 | 样式细节、文案、间距、图标、轻微体验问题 |

## KI-003：表单校验错误不展示且提交按钮卡住

| 字段 | 内容 |
|---|---|
| 问题编号 | KI-003 |
| 问题描述 | 多个前端表单填写错误时页面不提示具体错误，提交按钮一直卡住；浏览器控制台可看到 Zod 返回。 |
| 复现步骤 | 1. 打开登录、注册、新增乘机人、管理端编辑等表单；2. 输入非法邮箱、手机号、身份证号、缺失必填项或非法数值；3. 点击提交；4. 观察页面错误提示和按钮状态。 |
| 预期结果 | 页面展示字段级错误或全局错误；提交按钮在校验失败后立即恢复可点击；不要求用户打开控制台排错。 |
| 实际结果 | 已复现。在 `/booking/110124` 的新增乘机人弹窗中，空表单点击“确认添加”后页面不展示字段级错误，按钮变为“添加中...”且未恢复；浏览器控制台抛出 Zod 校验错误数组，包含 `name`、`idCardNo`、`phone` 的错误信息。 |
| 影响范围 | 用户认证、乘机人、预订、管理端表单等多个交互页面。 |
| 严重级别 | S1 |
| 当前状态 | 已复现；已确认新增乘机人表单，其他表单待扩展验证 |
| 建议优先级 | P1 |
| 后续验证方式 | 建立表单错误矩阵，逐页输入非法数据，记录字段提示、全局提示、按钮 loading 恢复和控制台报错。 |
| 关联 Issue | https://github.com/yunluoxincheng/skybooker-flight/issues/81 |
