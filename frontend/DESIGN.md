# DESIGN.md：SkyBooker 云航智订 UI/UX 设计规范

> 本文档用于指导 AI 或前端开发者生成本项目的用户界面。AI 生成页面时必须遵守本文档中的视觉风格、页面结构、组件规范和交互约束。

## 0. 品牌名称

- 产品中文名：云航智订
- 产品英文名：SkyBooker
- Logo 文案建议：SkyBooker
- 首页主标题建议：云航智订 SkyBooker
- 首页副标题建议：智能航班查询、AI 推荐、在线选座与机票预订

AI 生成 UI 时，页面标题、Header Logo、Hero 区域和后台系统名称应统一使用 “SkyBooker 云航智订”。

## 1. 项目 UI 定位

本项目是一个现代化航班查询与机票预订系统，整体界面应该体现：

- 专业；
- 清爽；
- 可信赖；
- 信息清晰；
- 购票流程明确；
- 有真实在线购票平台的感觉。

参考气质：

- 携程；
- 飞猪；
- 航旅纵横；
- 航空公司官网；
- 企业级 SaaS 管理后台。

不要做成：

- 过度花哨的旅游宣传页；
- 暗黑赛博朋克风；
- 纯表格堆砌页面；
- 颜色混乱的学生作业风。

## 3. 视觉风格

### 主色

使用蓝色作为主色，体现航空、科技、可信赖。

建议：

```text
主色：blue / sky / indigo
背景：slate-50 / white
文字：slate-900 / slate-600
边框：slate-200
强调：orange 用于价格，green 用于成功状态，red 用于取消和错误
```

### 卡片风格

- 圆角：`rounded-2xl`
- 阴影：`shadow-sm` 或 `shadow-md`
- 边框：`border border-slate-200`
- 背景：白色或轻微渐变

### 页面宽度

用户端主要内容：

```text
max-w-7xl mx-auto px-4 sm:px-6 lg:px-8
```

后台管理：

```text
左侧 Sidebar + 顶部 Header + 内容区
```

## 5. 首页设计

路径：`/`

首页目标：让用户快速开始查询航班或进入 AI 智能购票助手。

页面结构：

1. Hero 区域
   - 标题：云航智订 SkyBooker
   - 副标题：快速查询航班、智能推荐、在线选座与预订
   - 主按钮：立即查询航班
   - 次按钮：试试智能购票助手

2. 航班搜索卡片
   - 出发城市
   - 到达城市
   - 出发日期
   - 搜索按钮

3. 热门航线
   - 上海 → 北京
   - 广州 → 上海
   - 深圳 → 成都

4. 功能亮点
   - 智能推荐
   - 可视化选座
   - 候补购票
   - 在线退改签

## 6. 航班查询页设计

路径：`/flights`

页面目标：用户可以根据条件筛选航班，并进入详情或预订。

页面结构：

```text
顶部搜索栏
↓
左侧筛选栏 + 右侧航班列表
```

### 搜索栏

包含：

- 出发城市；
- 到达城市；
- 出发日期；
- 搜索按钮。

### 筛选栏

包含：

- 航空公司；
- 起飞时间段；
- 价格区间；
- 航班状态；
- 是否直飞；
- 行李额。

### 排序栏

包含：

- 综合推荐；
- 价格低优先；
- 飞行时间短优先；
- 起飞时间早优先；
- 余票多优先。

### 航班卡片

用户端不要优先使用普通表格，必须使用航班卡片。

卡片包含：

- 航空公司名称；
- 航班号；
- 出发城市与到达城市；
- 出发机场与到达机场；
- 起飞时间与到达时间；
- 飞行时长；
- 航班状态 Badge；
- 剩余座位；
- 价格；
- 查看详情按钮；
- 立即预订按钮。

## 7. AI 智能购票助手页面设计

路径：`/ai-assistant`

页面目标：用户通过自然语言表达购票需求，系统返回真实可购买航班推荐。

### 页面结构

```text
页面标题和说明
↓
聊天消息区域
↓
快捷示例按钮
↓
底部输入框
```

### 欢迎语

页面初始展示：

```text
你好，我是智能购票助手。你可以告诉我：
- 我想明天从上海去北京
- 帮我找便宜一点的航班
- 我想上午出发，不想中转
- 查询南方航空去广州的机票
```

### 推荐卡片

AI 返回航班时，在消息下方展示航班推荐卡片。

卡片内容：

- 航空公司；
- 航班号；
- 出发城市；
- 到达城市；
- 起飞时间；
- 到达时间；
- 飞行时长；
- 价格；
- 剩余座位；
- 航班状态；
- 查看详情按钮；
- 立即预订按钮。

底部提供：

- 查看全部结果按钮。

跳转规则：

```text
查看详情：/flights/[id]
立即预订：/booking/[flightId]
查看全部结果：/flights?departureCity=上海&arrivalCity=北京&departureDate=2026-05-26&sort=price_asc
```

## 8. 航班详情页设计

路径：`/flights/[id]`

页面结构：

1. 航班摘要；
2. 时间轴式起降信息；
3. 舱位价格；
4. 座位余量；

## 9. 预订与选座页设计

路径：`/booking/[flightId]`

页面目标：用户完成从确认航班到提交订单的流程。

页面结构：

1. 步骤条；
2. 航班摘要卡片；
3. 乘机人选择；
4. 可视化座位图；
5. 价格明细；
6. 提交订单按钮。

步骤条：

```text
确认航班 → 选择乘机人 → 选择座位 → 确认订单 → 模拟支付
```

座位图状态：

```text
可选：绿色或浅色边框
已售：红色或灰色禁用
锁定中：黄色
不可选：灰色
当前选择：蓝色高亮
```

## 10. 我的订单页设计

- 顶部订单状态 Tabs；
- 订单卡片列表；
- 每个订单显示航班、乘客、金额、状态、操作按钮。

订单操作：

- 继续支付；

## 11. 管理后台设计

后台页面采用企业管理系统风格。

管理后台入口为 `/admin`。该路径是前端页面路由，不是后端 JSON API。未登录时该入口展示管理员登录页，只包含管理员用户名、密码和登录按钮；登录成功后进入 `/admin/dashboard`。普通用户不能登录管理后台，管理员也不能通过用户端登录页进入用户端。所有后台 API 仍使用 `/api/admin/**`。

### 后台布局

```text
左侧 Sidebar
顶部 Header
右侧内容区
```

### 数据看板

路径：`/admin/dashboard`

包含：

- 今日订单数；
- 今日销售额；
- 退票数量；
- 候补订单数量；
- 热门航线图表；
- 订单状态分布图。

### 航班管理

路径：`/admin/flights`

使用表格展示：

- 航班号；
- 航空公司；
- 出发地；
- 目的地；
- 起飞时间；
- 价格；
- 状态；
- 操作。

操作：

- 新增；
- 编辑；
- 上架；
- 下架；
- 生成座位。

### 订单管理

路径：`/admin/orders`

使用 DataTable + Drawer 展示订单详情。

## 12. 响应式要求

桌面端优先，但需要适配移动端。

移动端规则：

- 筛选栏变成 Drawer；
- 航班卡片单列；
- 后台管理可仅保证桌面端体验；
- AI 聊天输入框固定底部。

## 13. 动效要求

可以使用轻微动效：

- 按钮 hover；
- 卡片 hover 阴影；
- 页面切换轻微 fade；
- 消息出现动画。

不要使用过度复杂动画。

## 14. AI 生成 UI 的强约束

AI 生成 UI 时必须遵守：

1. 不要生成暗黑赛博朋克风格。
2. 不要使用过度花哨的渐变背景。
3. 用户端航班列表必须使用航班卡片，不要只用表格。
4. 后台管理可以使用表格。
5. 航班查询页必须包含搜索栏、筛选栏、排序栏、航班列表。
6. AI 助手必须是聊天界面 + 航班推荐卡片。
7. 预订页必须包含步骤条、乘机人选择、座位图、价格明细。
8. 所有主要操作按钮必须清晰。
9. 页面必须桌面端优先，兼容移动端。
10. 组件必须基于 shadcn/ui + Tailwind CSS 实现。

## 认证页面 UI 设计

认证页面包括：

```text
/login
/register
/forgot-password
```

### 认证页整体风格

采用左右分栏布局：

- 左侧：品牌展示区，展示 “SkyBooker 云航智订”、产品副标题、航空插画或渐变背景；
- 右侧：白色表单卡片，包含登录、注册或找回密码表单。

移动端改为单列布局，品牌展示区缩小为顶部标题。

### 登录页

表单字段：

- 邮箱输入框；
- 密码输入框；
- 登录按钮；
- “忘记密码？”链接；
- “还没有账号？立即注册”链接。

### 注册页

表单字段：

- 邮箱输入框；
- 发送验证码按钮；
- 邮箱验证码输入框；
- 昵称输入框；
- 密码输入框；
- 确认密码输入框；
- 注册按钮。

交互要求：

- 发送验证码按钮点击后显示 60 秒倒计时；
- 验证码输入框限制 6 位数字；
- 密码强度不足时给出提示；
- 注册成功后显示成功提示并跳转登录页。

### 忘记密码页

表单字段：

- 邮箱输入框；
- 发送验证码按钮；
- 验证码输入框；
- 新密码输入框；
- 确认新密码输入框；
- 重置密码按钮。

### 设计限制

- 不要把微信登录、支付宝登录作为主要按钮；
- 如果展示第三方登录，只能放在扩展区域，并标注“后续扩展”；
- 不要要求用户填写手机号作为必填项；
- 主流程必须围绕邮箱验证码和邮箱密码登录设计。
