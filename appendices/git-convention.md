# Git 提交规范

## 1. 分支建议

```text
main        稳定版本
dev         开发分支
feature/*   功能分支
fix/*       修复分支
```

## 2. 提交格式

```text
<type>: <description>
```

## 3. type 类型

| 类型 | 说明 |
|---|---|
| feat | 新功能 |
| fix | 修复 Bug |
| docs | 文档 |
| style | 样式调整 |
| refactor | 重构 |
| test | 测试 |
| chore | 构建或工具配置 |

## 4. 示例

```text
feat: 新增航班查询接口
feat: 实现 AI 智能购票助手页面
fix: 修复座位并发锁定问题
docs: 更新部署指南
refactor: 重构订单服务
```
